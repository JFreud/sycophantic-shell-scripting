What is Shell Scripting?

A shell script is a program designed to be run by a Unix shell. It contains a sequence of commands that can be used to edit files, access data, and other things designed for the terminal. Because it was designed to be run in a shell, the syntax for a shell script is the same as the syntax for the command line. The first Unix shell was invented in 1971 by Ken Thompson at Bell Labs.

Can be useful for:

  • Administrative tasks
    • e.g. disk backups, system logs
  • Cute small programs
    • e.g. checking the weather, finding out what song you‚Äôre currently listening to
  • Program interactions
    • e.g. piping between files
  • Startup scripts
    • e.g. small programs to prototype application ideas for the startup

Pros/Cons?

Pros:

  • Easy syntax (same as command line)
  • Writing shell code is often quicker than writing the same code in other languages
  • Closer integration with OS
  • Widely installed (because the only thing necessary to run a shell script is a Unix shell)

Cons:

  • Relatively slow runtime
  • Easy to make costly mistakes
    • e.g. cd vs cp, < (write to a file) vs << (append to a file)
  • Can‚Äôt perform complex tasks (since it's limited by what the command line can do)
  • There can be a lack of compatibility between systems
    • e.g. Windows command line is different than Lenovo

Syntax

Useful Characters

  • # - comment
  • ; - command separator
  • $ - variable substitution
  • (()) - arithmetic expressions
  • () - subshell
  • All the ones you use in the command line (pipes, asterisks, gang signs, etc)

Variables

  • Variable assignment
    • num=3 (no space)
  • Read variables using $
    • echo ‚Äú$num‚Äù (not single quote)
    • substitution occurs before command is run
  • Command line arguments are $1-9
    • cp $1 $2 copies from the first argument to the second
  • variable=$( command ) saves output of command to variable
  • read input reads users input into variable input

Arithmetic

  • Do math using let
    • let ‚Äúa = 5 + 4‚Äù
    • let a=5+4 (can‚Äôt have spaces when not using quotes)
  • You can use double brackets $(( expression )) format
    • a=$(( $a + 1 ))
    • (( a ++ ))

If Statements

  • if [ <conditional> ]
  • then
    • <commands>
  • else
    • <commands>
  • fi
  • [ or test is the condition evaluation utility--it returns 0 if true and 1 if false
  • fi ends the if statement
  • you can add an elif and another then
  • Boolean operators are && and ||
